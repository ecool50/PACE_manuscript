# PACE: manuscript analysis code

Code and reproduction materials for the manuscript

> *PACE: proximity-associated changes in expression for imaging-based spatial transcriptomics*

This repository contains everything needed to reproduce the analyses, figures, and
supplementary materials of the PACE manuscript, for both the breast cancer (Xenium)
and melanoma (CosMx) cohorts, including the method comparison against SpatioMark and
niche-DE.

For the **PACE R package itself**, see
[github.com/ecool50/PACE](https://github.com/ecool50/PACE) (source) or
[ecool50.github.io/PACE](https://ecool50.github.io/PACE/) (function reference +
vignette). The package and this repository are versioned together.

---

## Quick start

```bash
git clone https://github.com/ecool50/PACE_manuscript.git
cd PACE_manuscript

# R + Bioconductor deps, the PACE package, and the comparison methods
Rscript setup_environment.R

# the two raw inputs (~0.6 GB): melanoma from the public SIMVI Zenodo
# record, breast cancer from the author deposit (see fetch_data.R)
Rscript fetch_data.R

# render the primary breast cancer analysis (Figure 2)
quarto render pace_mv/notebooks/analysis_bc.qmd
```

Every analysis is driven by the [PACE package](https://github.com/ecool50/PACE):
each notebook refits PACE from the raw data with `paceFit()` and regenerates its
figures with the package's plotting functions. A fit takes a few minutes on a
standard workstation.

## Prerequisites

- **R 4.5.0** (older/newer 4.x may work, but 4.5.0 is the verified `sessionInfo()`).
- A **C/C++ toolchain** (`xcode-select --install` on macOS, `build-essential` on
  Debian/Ubuntu): the PACE contamination solver and `TMB`/`Rcpp` compile from source.
- **Quarto** ([quarto.org](https://quarto.org)) to render the `.qmd` notebooks.
- A working **Python** for `zellkonverter`/`basilisk` (only the melanoma notebook,
  which reads an `.h5ad`); the first run provisions this automatically.

## Data

The two raw inputs are too large for git; `fetch_data.R` downloads and
md5-verifies each into the paths the notebooks read from:

| Path | Contents | Source |
|------|----------|--------|
| `data/simvi_melanoma/14708000/Melanoma_5612.h5ad` | Melanoma CosMx cohort | Public SIMVI record [10.5281/zenodo.14708000](https://doi.org/10.5281/zenodo.14708000) (Dong et al., CC BY 4.0) — fetched automatically |
| `data/spe_10x_nuclei_withMetrics.rds` | Breast cancer Xenium `SpatialExperiment` (10x public data, nucleus segmentation, scClassify cell-type annotation) | Author deposit — set `BC_URL`/`BC_MD5` in `fetch_data.R` (see `ZENODO.md`) |

These two inputs are all the notebooks require; PACE fits and the
competitor-method results are regenerated (and cached under `data/`) on first
render. The melanoma download needs no configuration; only the breast cancer
`BC_URL`/`BC_MD5` must be filled in before first use.

## Which notebook makes which figure

| Notebook | Manuscript output |
|----------|-------------------|
| `pace_mv/notebooks/analysis_bc.qmd` | **Figure 2** (macrophage–tumour panel: decomposition, pairwise heatmap, drivers, proximity, ROI maps); supplementary anchor-gene tables |
| `pace_mv/notebooks/analysis_mel.qmd` | **Figure 4** (macrophage SPP1 response panel); melanoma anchor / radius tables |
| `pace_mv/notebooks/nichede_bc.qmd` | niche-DE driver rankings (input to the Figure 3 comparison) |
| `pace_mv/notebooks/spatiomark_bc.qmd` | SpatioMark driver rankings (input to the Figure 3 comparison) |
| `pace_mv/notebooks/spillover_composite_bc.qmd` | **Figure 3** (contamination method comparison); supplementary concordance and radius/runtime figures |

Figure 1 is a conceptual schematic and is not generated by a notebook.

## Repository layout

```
PACE_manuscript/
  setup_environment.R      # install all dependencies + the PACE package
  fetch_data.R             # download the analysis data from Zenodo
  pace_mv/
    notebooks/             # the five analysis notebooks above
    helpers/               # repo-local helpers: ROI-inset figure + DenoIST correction
  data/                    # fetched (raw SPE + h5ad), never committed
```

The PACE method itself (model fitting, contamination correction, shrinkage,
variance decomposition, driver scores, and the standard figures) lives entirely
in the [PACE package](https://github.com/ecool50/PACE). This repository only adds
the cohort-specific data preparation, the bespoke ROI-inset figure, and the
competitor methods (niche-DE, SpatioMark, DenoIST).
